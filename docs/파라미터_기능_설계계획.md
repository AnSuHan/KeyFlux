# 파라미터 기능 설계 계획 (재사용 변수 + 인라인 인자)

> **상태: v1.0.5 에서 구현 완료.** 아래 계획대로 재사용 변수 + 인라인 위치 인자
> (`{1}`/`{2}`/`{*}`/`{1:기본값}`/`{이름}`, 이스케이프 `{{`/`}}`)를 구현했다.
> 열린 결정(§9)은 추천값(Enter 종료자, `{*}`, 인자 없으면 빈값, special 트리거
> 한정)으로 확정. 이 문서는 설계 근거 기록용으로 남긴다.

단축어 출력에 **파라미터**를 쓸 수 있게 하는 기능의 설계 계획서다.

> 결정된 방향(사용자 선택):
> - **재사용 변수 + 호출 시 인라인 인자 둘 다** 지원
> - 인자는 **트리거 뒤에 이어서 입력(인라인)** 방식

관련 현재 코드(모두 `main.py`):
치환 = `resolve_output()` + `_PLACEHOLDER_RE`(≈560), 트리거 발동 = special
`_check_immediate`(≈1002)·word/regex `_check_and_replace`(≈1069),
실제 치환(백스페이스+재타이핑) = `_do_replace`(≈1135).

---

## 1. 최종 동작 (사용자 관점 사양)

출력 템플릿 플레이스홀더를 하나의 문법으로 통합한다.

| 문법 | 의미 | 예 |
|---|---|---|
| `{date}` `{time}` `{datetime}` (`:형식`) | 날짜/시간 (기존 유지) | `{date:%m-%d}` |
| `{1}` `{2}` … | 호출 시 **위치 인자**(공백으로 구분된 n번째) | `{1}` |
| `{*}` | **인자 전체**(내부 공백 보존) | `{*}` |
| `{이름}` | 미리 정의한 **재사용 변수** | `{회사}` |

**대표 흐름** (변수 `회사=한빛`, 규칙 `트리거=;메일`, `출력={회사} {1} 드림`):

```
입력:  ;메일 홍길동⏎
결과:  한빛 홍길동 드림
```

- 트리거 뒤 **공백**으로 인자 시작, **Enter**로 확정(=인자 캡처 종료, Enter는 소비).
- `{1}`/`{*}` 같은 **인자 플레이스홀더가 있는 규칙만** "인자 대기" 모드가 된다.
  `{회사}`만 있는 규칙은 즉시 정적 치환(대기 없음).

## 2. 데이터 모델 / 저장

- 신규 저장소 `keyflux_variables.json` (rules/settings 파일 옆, `SETTINGS_FILE`

- 변수명 규칙: 식별자, `date/time/datetime` 예약어 및 **순수 숫자 금지**
  (숫자는 위치 인자와 충돌). 저장 시 검증.

## 3. 치환 엔진 (`resolve_output` 확장, `main.py:560-578`)

- `_PLACEHOLDER_RE` 를 `\{([^{}:]+)(?::([^{}]*))?\}` 로 확장(키를 일반화).
- `resolve_output(output, args=None, variables=None)` 로 시그니처 확장
  (기본값 None → 기존 동작 100% 호환).
- **해석 우선순위**: ① `date/time/datetime` → ② 숫자 키(위치 인자) →
  ③ `*`(전체 인자) → ④ 변수명 → ⑤ 미해석 시 그대로/빈값.
- 부가 결정거리(§9): `{1:기본값}`(인자 없을 때 기본값), 리터럴 중괄호
  `{{`/`}}` 이스케이프.

## 4. 런타임 인자 캡처 (핵심 — 상태머신)

현재 special 트리거는 완성 즉시 발동(`_check_immediate` `main.py:1002`).
파라미터 규칙은 **즉시 발동을 보류하고 인자를 모아야** 한다.

- **규칙 로드 시**: 출력에 `{\d+}`/`{\*}` 포함 여부로 `parameterized` 플래그
  계산(`_rebuild_indexes` `main.py:866`에서).
- **트리거 완성 시**: `parameterized`면 `_fire_special` 대신 캡처 상태 진입 —
  `self._capturing = {rule, trigger_end_pos, ...}` (버퍼 내 트리거 끝 위치 기록).
- **캡처 중 키 입력**(`_on_press` `main.py:956`):
  - 일반 글자/공백 → 버퍼에 계속 누적(화면엔 그대로 보임, suppress=False).
  - **Enter(종료자)** → 확정: `args_raw = compose_hangul(buffer[trigger_end:])`
    에서 선행 공백 1개 제거 → 공백 split → `{1}`,`{2}`… / `{*}`=전체.
    `resolve_output(출력, args, variables)` → 스팬 치환.
  - **Esc / 길이 상한(예: 100자) / 트리거 무관 특수키** → 캡처 취소(입력 그대로).
    오작동으로 타이핑이 하이재킹되지 않도록 **안전장치 필수**.
- **다른 매칭 일시정지**: 캡처 중에는 다른 트리거 매칭을 멈추고 누적만.

## 5. 삭제 스팬 정확성 (`_do_replace` 재사용, `main.py:1135`)

- 최근에 고친 **before/after 조합 비교 방식**을 그대로 확장: 지울 스팬이 이제
  `트리거 + 공백 + 인자`(+종료자 Enter). `consumed_raw` 를 "트리거 시작
  위치"까지로 잡으면 기존 로직이 **경계 병합 보정 포함** 그대로 동작한다.
- 종료자 Enter 는 `extra`("\n")로 넘겨 화면에서 함께 삭제(단, 출력이 그 자리를
  대체하므로 재타이핑엔 미포함이 자연스럽다 — 세부는 §9).

- **RuleDialog**(`main.py:1289`): 힌트/미리보기에 `{1}`,`{*}`,`{변수}` 안내 추가.
  미리보기는 **샘플 인자**(`{1}`→"인자1")와 **현재 변수값**을 대입해 결과 형태를
  보여준다. `{1}` 포함 시 "이 규칙은 호출 시 인자를 받습니다" 배지.
- **설정 창**: "변수" 관리 테이블(이름 | 값, 추가/수정/삭제) 추가 → `variables`
  저장소에 반영, 리스너에 전파.

## 7. 기존 기능과의 상호작용 (검토 필요 지점)

- **정규화(한/영)**: 트리거 매칭은 기존대로. **인자는 리터럴**(조합형
  `compose_hangul`으로 화면 그대로 삽입, 정규화 안 함).
- **접두사 겹침/보류**(`_pending`, `_holdable_rivals` `main.py:917`): 파라미터
  트리거가 다른 트리거의 접두사면 캡처 시작 시점과 보류 로직이 충돌 가능 →
  규칙 정의: 캡처는 "완전 일치 + parameterized"에서만 시작, 라이벌 있으면
  기존 보류 우선.
- **변환 중 차단**(win32_event_filter): 캡처는 주입 이전 단계라 무관(안전).
  ([[키입력_처리와_블로킹_정책]] 참고)
- **다중 줄 출력 + Enter 종료자**: Enter를 인자 종료자로 쓰므로, 파라미터
  규칙의 출력에 줄바꿈은 `{...}` 결과나 저장된 `\n`으로만(사용자가 인자 입력
  중 Enter는 종료). 문서화 필요.

## 9. 정하면 좋은 열린 결정거리

1. **인자 종료자**: `Enter`(추천, 단순) vs 전용 토큰 `;;` vs 이중 스페이스.
   → 다중 줄 출력과의 상충 때문에 중요.
2. **인자 없음 처리**: 빈 문자열 vs `{1:기본값}` 문법 지원.
3. **`{*}` 표기**: `{*}` vs `{@}` vs `{0}`.
4. **리터럴 중괄호 이스케이프**: `{{`/`}}` 지원 여부.
5. **word/regex 트리거 인자 지원**: 1차는 special만(권장), 이후 확장.

## 10. 단계적 도입 (권장)

- **Phase 1 — 재사용 변수**(정적, 저위험): `resolve_output`+변수 저장소+설정 UI.
  런타임 캡처 없음 → 먼저 배포.
- **Phase 2 — 인라인 위치 인자**(special): 캡처 상태머신 + 스팬 삭제 + 안전장치.
- **Phase 3(선택)**: 기본값/이스케이프/`{*}`/word·regex 확장.
