# KeyFlux — 키 입력 자동 변환기

> **v1.0.3**

등록한 트리거(단어/단축어/정규식)를 타이핑하면 지정한 텍스트로 자동
치환해 주는 백그라운드 도구입니다. 메모장·브라우저뿐 아니라 터미널
(PowerShell/cmd/Git Bash)에서도 동작합니다.

## 주요 기능

- **트리거 타입 3종**: `word`(단어), `special`(`;`단축어), `regex`(정규식)
- **특수 변수 치환**: `{date}` `{time}` `{datetime}` (형식 지정: `{date:%Y%m%d}`)
- **한/영·대소문자 무관 매칭** (v1.0.2, 기본 활성화): 트리거를 등록한
  입력 모드(한/영)나 대소문자와 상관없이 **같은 물리 키**를 누르면 동작.
  예) 영문 `abc` 트리거를 한글 모드로 같은 키를 쳐도(→`ㅁㅠㅊ`) 변환됨.
  **설정(⚙) 창**의 **한/영·대소문자 무관 매칭** 체크박스로 끌 수 있음.
- **접두사 겹침 트리거 지원** (v1.0.2): `::d` 와 `::dev` 처럼 짧은 트리거가
  긴 트리거의 접두사여도 둘 다 사용 가능 (긴 트리거 완성 시 그쪽을, 더
  길게 갈 수 없으면 짧은 쪽을 확정).
- **규칙 편집창에서 Enter 로 저장** (v1.0.2)
- **`;;` 규칙 1순위 정렬** (v1.0.3): 트리거가 `;;` 로 시작하는 규칙을 목록
  맨 위로 항상 정렬(같은 그룹 내 순서는 유지).
- **시작 시 자동 실행 / 윈도우 알림 표시 / 디버그 로그** 토글 (설정 창)
- **결과 대소문자 보장** 옵션 (기본 활성화, 설정 창): CapsLock 이
  켜져 있어도 출력 결과의 대소문자가 저장한 그대로 들어감. 매칭 옵션과
  **별개로 on/off** 가능(끄면 CapsLock 상태에 따라 대소문자가 뒤집힐 수 있음).
- **설정 창**: 위 동작 옵션(매칭·대소문자·알림·시작 시 자동 실행·디버그)은
  상단 툴바 **⚙ 설정** 버튼으로 여는 별도 설정 창에서 한곳에 모아 조작.
- 트레이 상주, 드래그로 규칙 순서 변경, JSON 내보내기(파일명 타임스탬프 자동)/불러오기

## 설정 파일 (rules.json)

### 내보내기
앱 GUI 상단 툴바 **↑ 내보내기** 버튼 → 원하는 경로에 JSON 저장

### 불러오기
**↓ 불러오기** 버튼 → JSON 선택 → 대체 또는 추가 선택

### 직접 편집
```json
[
  { "type": "word",    "trigger": "abc",    "output": "123",    "enabled": true },
  { "type": "special", "trigger": ";date",  "output": "{date}", "enabled": true },
  { "type": "regex",   "trigger": "\\d{4}", "output": "[YEAR]", "enabled": false }
]
```

특수 변수: `{date}` `{time}` `{datetime}`

---

## 설정 파일 우선순위

| 위치 | 설명 |
|------|------|
| 실행파일(또는 스크립트) 옆 `rules.json` | **최우선** — 동봉 배포용 |
| `~/.keyflux_rules.json` | fallback — 개인 설정 |

동작 옵션(한/영·대소문자 무관 매칭, 결과 대소문자 보장, 알림 표시, 시작 시
자동 실행, 디버그 로그 등 on/off)은 규칙 파일과 별개로 `rules.json` 옆
`keyflux_settings.json`(없으면 `~/.keyflux_settings.json`)에 저장됩니다.

## 시작 프로그램 등록 (Windows 로그인 시 자동 실행)

Windows 에 로그인하면 KeyFlux 가 자동으로 떠서 트레이에 상주하도록 등록할
수 있습니다.

### 앱에서 등록/해제 (권장)

1. 상단 툴바 **⚙ 설정** 버튼을 눌러 설정 창을 엽니다.
2. **알림 · 시작** 그룹의 **시작 시 자동 실행** 체크박스를 켜면 등록,
   끄면 해제됩니다.

내부적으로는 현재 사용자 레지스트리
`HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run` 에
`KeyFlux` 값으로 실행 명령을 등록합니다(관리자 권한 불필요). exe 로 실행
중이면 그 exe 경로가, 스크립트로 실행 중이면 `python main.py` 명령이
등록됩니다. 등록 위치가 사용자 단위이므로 **로그인한 본인 계정에서만**
자동 실행됩니다.

### 수동으로 확인/해제하려면

- **시작 프로그램 폴더 방식이 아닌 레지스트리 방식**이므로, `Win+R` →
  `regedit` → 위 `Run` 키에서 `KeyFlux` 값을 직접 지워도 해제됩니다.
- 또는 **작업 관리자 → 시작프로그램 탭**에서 `KeyFlux` 항목을
  "사용 안 함" 으로 바꿀 수 있습니다.

## exe로 빌드하기 (Windows)

### 1. PyInstaller 설치

가상환경을 켠 상태에서:

```powershell
pip install -r requirements.txt
pip install pyinstaller
```

### 2. 빌드 실행

```powershell
python build.py
```

빌드가 끝나면 **버전이 포함된** `dist\KeyFlux_v<버전>.exe`(예: `dist\KeyFlux_v1.0.3.exe`)가 생성됩니다. 이 파일 하나만 복사해서 Python이 설치되지 않은 PC에서도 바로 실행할 수 있습니다. (GitHub 릴리스에 올리는 실행파일명에는 항상 버전을 포함합니다.)

### 3. 특정 규칙(rules.json)을 포함해서 배포하기

미리 원하는 규칙으로 구성한 `rules.json`을 `main.py`와 같은 폴더에 둔 뒤:

```powershell
python build.py --with-rules
```

`dist\KeyFlux_v<버전>.exe`와 `dist\rules.json` **두 파일이 함께 생성**됩니다. 배포할 때 이 두 파일을 같은 폴더에 넣어서 전달하세요 (exe만 옮기면 기본 규칙으로 시작됩니다).

### 4. 직접 PyInstaller 명령으로 빌드하고 싶다면

```powershell
pyinstaller --onefile --windowed --name KeyFlux_v1.0.3 ^
  --icon keyflux.ico --add-data "keyflux.ico;." main.py
```

> `--icon` 은 exe 파일/작업표시줄 파비콘을, `--add-data "keyflux.ico;."` 는
> 실행 중 창/트레이 아이콘이 같은 .ico 를 로드하도록 onefile 에 동봉합니다.
> (둘을 같이 줘야 파일 아이콘과 실행 중 아이콘이 동일해집니다.) 보통은
> `python build.py` 가 이 옵션들을 자동으로 붙여 줍니다.

### 빌드 후 체크리스트

- [ ] `dist\KeyFlux_v<버전>.exe` 더블클릭 시 정상 실행되는지 확인
- [ ] 트레이/작업표시줄 아이콘이 `keyflux.ico` 와 동일하게 나타나는지 확인
- [ ] **Windows Defender/백신이 "알 수 없는 앱"으로 경고**할 수 있음 — 키 입력을 감지하는 프로그램 특성상 흔한 현상입니다. "추가 정보 → 실행" 또는 백신 예외 처리 필요
- [ ] `--with-rules`로 빌드했다면 exe와 rules.json이 같은 폴더에 있는지 확인
