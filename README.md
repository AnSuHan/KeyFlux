# KeyFlux — 키 입력 자동 변환기

> **v1.0.5**

등록한 트리거(단어/단축어/정규식)를 타이핑하면 지정한 텍스트로 자동
치환해 주는 백그라운드 도구입니다. 메모장·브라우저뿐 아니라 터미널
(PowerShell/cmd/Git Bash·**VSCode 통합 터미널**)에서도 동작합니다.

## 주요 기능

- **트리거 타입 3종**: `word`(단어), `special`(`;`단축어), `regex`(정규식)
- **특수 변수 치환**: `{date}` `{time}` `{datetime}` (형식 지정: `{date:%Y%m%d}`)
- **호출 시 인자 + 재사용 변수** (신규): 출력에 `{1}` `{2}` `{*}`(위치 인자)와
  `{이름}`(재사용 변수)을 넣어 파라미터화. `special` 트리거를 친 뒤 **공백으로
  인자를 이어 입력하고 Enter 로 확정**하면 인자가 대입됨. 변수는 **설정 창의
  변수 관리**에서 추가/수정, 규칙 편집창의 **? 문법 도움말** 버튼에서 전체
  문법 확인 가능. (예: `;메일`→`{회사} {1} 드림`, 입력 `;메일 홍길동⏎` →
  `한빛 홍길동 드림`)
- **한/영·대소문자 무관 매칭** (v1.0.2, 기본 활성화): 트리거를 등록한
  입력 모드(한/영)나 대소문자와 상관없이 **같은 물리 키**를 누르면 동작.
  예) 영문 `abc` 트리거를 한글 모드로 같은 키를 쳐도(→`ㅁㅠㅊ`) 변환됨.
  **설정(⚙) 창**의 **한/영·대소문자 무관 매칭** 체크박스로 끌 수 있음.
- **접두사 겹침 트리거 지원** (v1.0.2): `::d` 와 `::dev` 처럼 짧은 트리거가
  긴 트리거의 접두사여도 둘 다 사용 가능 (긴 트리거 완성 시 그쪽을, 더
  길게 갈 수 없으면 짧은 쪽을 확정).
- **규칙 편집창에서 Enter 로 저장** (v1.0.2)
- **`;;` 규칙 1순위 정렬** (v1.0.3): 트리거가 `;;` 로 시작하는 규칙을 목록
  맨 위로 항상 정렬(같은 그룹 내 순서는 유지).
- **VSCode·Electron 터미널 호환** (v1.0.4): VSCode 통합 터미널처럼 Chromium
  기반 앱에서도 치환이 정상 입력됨(이전엔 입력이 안 됐음).
- **입력 언어(한/영) 보존** (v1.0.4): 치환이 끝나면 키워드를 입력하기 전의
  한/영 상태가 그대로 유지됨(출력이 한글이든 영어든 무관).
- **규칙 추가 시 단축어(`special`)가 기본·1순위** (v1.0.4): 새 규칙 추가 창에서
  타입 기본 선택이 `special`(`;`단축어).
- **시작 시 자동 실행 / 윈도우 알림 표시 / 디버그 로그** 토글 (설정 창)
- **결과 대소문자 보장** 옵션 (기본 활성화, 설정 창): CapsLock 이
  켜져 있어도 출력 결과의 대소문자가 저장한 그대로 들어감. 매칭 옵션과
  **별개로 on/off** 가능(끄면 CapsLock 상태에 따라 대소문자가 뒤집힐 수 있음).
- **설정 창**: 위 동작 옵션(매칭·대소문자·알림·시작 시 자동 실행·디버그)은
  상단 툴바 **⚙ 설정** 버튼으로 여는 별도 설정 창에서 한곳에 모아 조작.
- 트레이 상주, 드래그로 규칙 순서 변경, JSON 내보내기(파일명 타임스탬프 자동)/불러오기

## 설정 파일 (rules.json)

### 내보내기
앱 GUI 상단 툴바 **↑ 내보내기** 버튼 → 원하는 경로에 JSON 저장

### 불러오기
**↓ 불러오기** 버튼 → JSON 선택 → 대체 또는 추가 선택

### 직접 편집
```json
[
  { "type": "word",    "trigger": "abc",    "output": "123",    "enabled": true },
  { "type": "special", "trigger": ";date",  "output": "{date}", "enabled": true },
  { "type": "regex",   "trigger": "\\d{4}", "output": "[YEAR]", "enabled": false }
]
```

특수 변수: `{date}` `{time}` `{datetime}`

## 출력 문법 (플레이스홀더)

출력(`output`)에는 아래 문법을 쓸 수 있습니다. 규칙 편집창의 **? 문법 도움말**
버튼에서도 예시와 함께 볼 수 있습니다.

| 문법 | 의미 | 예 |
|------|------|-----|
| `{date}` `{time}` `{datetime}` | 날짜/시간 (콜론 뒤 strftime 형식) | `{date:%m-%d}` → `07-11` |
| `{1}` `{2}` … | 호출 시 **위치 인자**(공백으로 구분) | `{1}` |
| `{1:기본값}` | 그 인자가 없을 때 쓸 기본값 | `{1:고객}` |
| `{*}` | **인자 전체**(내부 공백 보존) | `{*}` |
| `{이름}` | 미리 정의한 **재사용 변수** | `{회사}` |
| `{{` `}}` | 리터럴 중괄호(`{` `}`) | `{{1}}` → `{1}` |

### 호출 시 인자 입력 방법
- `{1}`/`{*}` 를 포함한 **`special` 규칙**은 트리거를 완성해도 즉시 변환되지 않고
  **인자 입력을 기다립니다**.
- 트리거 뒤에 **공백으로 인자를 이어서 입력**하고, **Enter 로 확정**하면 변환됩니다.
  - 예) 트리거 `;메일`, 출력 `{1} 님께` → `;메일 홍길동⏎` → `홍길동 님께`
- `{1}`/`{*}` 가 없는 규칙(`{회사}` 등)은 **즉시** 변환됩니다(대기 없음).
- 인자 입력은 `special` 트리거에서만 동작합니다(`word`/`regex` 제외).
- Enter 는 인자 입력을 끝내는 **종료키**이므로, 출력에 줄바꿈을 넣으려면
  규칙 편집창 출력칸에서 직접 Enter 로 줄을 나누세요.

### 재사용 변수
- **설정(⚙) 창 → 변수 관리**에서 이름·값을 추가/수정/삭제합니다.
- 출력 어디에서나 `{이름}` 으로 참조합니다(파라미터 규칙 여부와 무관).
- 변수는 규칙 파일 옆 `keyflux_variables.json`(없으면 `~/.keyflux_variables.json`)에
  저장됩니다.
- 변수명 제약: `{ } :` 기호·순수 숫자·`*`·`date`/`time`/`datetime`(예약어) 불가.

## 시작 프로그램 등록 (Windows 로그인 시 자동 실행)

Windows 에 로그인하면 KeyFlux 가 자동으로 떠서 트레이에 상주하도록 등록할
수 있습니다.

### 앱에서 등록/해제 (권장)

1. 상단 툴바 **⚙ 설정** 버튼을 눌러 설정 창을 엽니다.
2. **알림 · 시작** 그룹의 **시작 시 자동 실행** 체크박스를 켜면 등록,
   끄면 해제됩니다.

내부적으로는 현재 사용자 레지스트리
`HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run` 에
`KeyFlux` 값으로 실행 명령을 등록합니다(관리자 권한 불필요). exe 로 실행
중이면 그 exe 경로가, 스크립트로 실행 중이면 `python main.py` 명령이
등록됩니다. 등록 위치가 사용자 단위이므로 **로그인한 본인 계정에서만**
자동 실행됩니다.

### 수동으로 확인/해제하려면

- **시작 프로그램 폴더 방식이 아닌 레지스트리 방식**이므로, `Win+R` →
  `regedit` → 위 `Run` 키에서 `KeyFlux` 값을 직접 지워도 해제됩니다.
- 또는 **작업 관리자 → 시작프로그램 탭**에서 `KeyFlux` 항목을
  "사용 안 함" 으로 바꿀 수 있습니다.

## exe로 빌드하기 (Windows)

### 1. PyInstaller 설치

가상환경을 켠 상태에서:

```powershell
pip install -r requirements.txt
pip install pyinstaller
```

### 2. 빌드 실행

```powershell
python build.py
```

빌드가 끝나면 **버전이 포함된** `dist\KeyFlux_v<버전>.exe`(예: `dist\KeyFlux_v1.0.5.exe`)가 생성됩니다. 이 파일 하나만 복사해서 Python이 설치되지 않은 PC에서도 바로 실행할 수 있습니다. (GitHub 릴리스에 올리는 실행파일명에는 항상 버전을 포함합니다.)

### 3. 특정 규칙(rules.json)을 포함해서 배포하기

미리 원하는 규칙으로 구성한 `rules.json`을 `main.py`와 같은 폴더에 둔 뒤:

```powershell
python build.py --with-rules
```

`dist\KeyFlux_v<버전>.exe`와 `dist\rules.json` **두 파일이 함께 생성**됩니다. 배포할 때 이 두 파일을 같은 폴더에 넣어서 전달하세요 (exe만 옮기면 기본 규칙으로 시작됩니다).

### 4. 직접 PyInstaller 명령으로 빌드하고 싶다면

```powershell
pyinstaller --onefile --windowed --name KeyFlux_v1.0.5 ^
  --icon keyflux.ico --add-data "keyflux.ico;." main.py
```

> `--icon` 은 exe 파일/작업표시줄 파비콘을, `--add-data "keyflux.ico;."` 는
> 실행 중 창/트레이 아이콘이 같은 .ico 를 로드하도록 onefile 에 동봉합니다.
> (둘을 같이 줘야 파일 아이콘과 실행 중 아이콘이 동일해집니다.) 보통은
> `python build.py` 가 이 옵션들을 자동으로 붙여 줍니다.

### 빌드 후 체크리스트

- [ ] `dist\KeyFlux_v<버전>.exe` 더블클릭 시 정상 실행되는지 확인
- [ ] 트레이/작업표시줄 아이콘이 `keyflux.ico` 와 동일하게 나타나는지 확인
- [ ] **Windows Defender/백신이 "알 수 없는 앱"으로 경고**할 수 있음 — 키 입력을 감지하는 프로그램 특성상 흔한 현상입니다. "추가 정보 → 실행" 또는 백신 예외 처리 필요
- [ ] `--with-rules`로 빌드했다면 exe와 rules.json이 같은 폴더에 있는지 확인

## 변경 이력

### v1.0.5
- **호출 시 인자(위치 인자) + 재사용 변수**: 출력에 `{1}` `{2}` `{*}`(위치 인자),
  `{이름}`(재사용 변수), `{1:기본값}`(기본값)을 넣어 파라미터화. `{1}`/`{*}` 가
  있는 `special` 규칙은 트리거를 친 뒤 **공백으로 인자를 이어 입력하고 Enter 로
  확정**하면 인자가 대입됨(예: `;메일 홍길동⏎` → `한빛 홍길동 드림`).
  `{이름}` 만 있는 규칙은 대기 없이 즉시 치환.
- **변수 관리 UI**: 설정(⚙) 창에 이름·값 테이블(추가/편집/삭제) 추가.
  변수는 `keyflux_variables.json` 에 저장.
- **문법 도움말 버튼**: 규칙 편집창·변수 관리 화면의 **? 문법 도움말** 버튼에서
  사용 가능한 모든 출력 문법을 예시와 함께 확인. 규칙 편집창은 파라미터 규칙
  **배지**와 샘플 인자/변수 대입 **실시간 미리보기** 표시.
- **리터럴 중괄호 이스케이프**: `{{` → `{`, `}}` → `}`.
- 인자 캡처는 기존 조합 경계 보정(문장 중간 사용 시 앞 글자/공백 보존) 로직을
  그대로 재사용하고, 종료자 Enter 는 화면에서 함께 삭제. 안전장치(Esc·트리거
  밖 백스페이스·길이 상한)로 캡처 취소.
- 단위 테스트 보완(resolve_output 인자/변수/기본값/이스케이프, 파라미터 감지,
  변수명 검증, 인자 캡처 상태머신·스크린 모델 등, 총 90개).

### v1.0.4
- **VSCode·Electron/Chromium 터미널 입력 수정**: VSCode 통합 터미널 등
  Chromium 기반 창은 레거시 `WM_IME_CONTROL`(IME 변환모드 제어)을 무시해,
  기존 "스캔코드+IME 강제" 방식이 안 먹혀 치환이 **아예 입력되지 않던** 문제를
  해결. 대상 창이 콘솔 줄 편집기(conhost/Windows Terminal/Git Bash 등)면
  스캔코드, 그 외 일반 GUI·Electron 이면 **유니코드 직접 주입**으로 분기.
- **입력 언어(한/영) 보존 강화**: 유니코드 직접 주입은 IME를 거치지 않아
  영문 출력이 한글로 바뀌지 않고, IME 모드를 건드리지 않으므로 **키워드
  입력 전의 한/영 상태가 그대로 유지**됨(콘솔은 기존처럼 IME 강제→복원).
- **규칙 추가 시 단축어(`special`)가 1순위·기본 선택**: 규칙 추가 창의 타입
  콤보 순서를 `special → word → regex` 로 변경.
- 단위 테스트 보완(주입 방식 분기·규칙 타입 순서, 총 47개).

### v1.0.3
- **설정 창 분리**: 동작 옵션(한/영·대소문자 무관 매칭, 결과 대소문자 보장,
  알림 표시, 시작 시 자동 실행, 디버그 로그)을 상단 툴바에서 빼고 **⚙ 설정**

  대소문자 그대로 주입. 매칭 옵션과 별개로 상단 툴바에서 on/off 가능.
